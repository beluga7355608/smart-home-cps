# 实验说明

## 1. 运行环境
- Windows 10/11
- Python 3.9+
- 需要管理员权限运行（raw socket 捕获/ICMP）

## 2. ARP（发送/解析）

### 2.1 发送ARP（触发解析）
```powershell
python "单元级CPS\arp_unit.py" send 192.168.1.1
```
说明：Windows 标准库无法直接发送以太网ARP帧，此命令使用 `SendARP` 触发ARP并输出MAC。

### 2.2 构造ARP帧并解析
```powershell
python "单元级CPS\arp_unit.py" build --src-mac 00:11:22:33:44:55 --src-ip 192.168.1.10 --dst-mac ff:ff:ff:ff:ff:ff --dst-ip 192.168.1.1
python "单元级CPS\arp_unit.py" parse --hex <上一步输出的HEX>
```

## 3. TCP报文段发送/接收

### 3.1 启动接收端
```powershell
python "单元级CPS\tcp_segment_demo.py" server --host 0.0.0.0 --port 9000
```

### 3.2 发送端（构造TCP段并作为数据发送）
```powershell
python "单元级CPS\tcp_segment_demo.py" client --host 127.0.0.1 --port 9000 --payload "hello tcp segment"
```

## 4. ICMP活动主机发现
```powershell
python "单元级CPS\icmp_scan.py" 192.168.1.1 192.168.1.254
```

## 5. 监控IP包流量（按源地址统计）
```powershell
python "单元级CPS\ip_traffic_monitor.py" --duration 20 --out ip_stats.csv
```

## 6. 解析IP数据包
```powershell
python "单元级CPS\ip_packet_parser.py" --count 10 --log ip_parse.log
```

## 7. TCP流分析（pcap）
```powershell
# 先用Wireshark保存pcap
python "单元级CPS\tcp_flow_analyzer.py" --pcap sample.pcap
# 指定流（输出某条连接的细节）
python "单元级CPS\tcp_flow_analyzer.py" --pcap sample.pcap --flow 192.168.1.10:51512-93.184.216.34:80
```

## 8. 自定义有序UDP协议

### 8.1 启动接收端
```powershell
python "单元级CPS\reliable_udp.py" recv --host 0.0.0.0 --port 9100
```

### 8.2 发送端
```powershell
python "单元级CPS\reliable_udp.py" send --host 127.0.0.1 --port 9100 --count 5
```
