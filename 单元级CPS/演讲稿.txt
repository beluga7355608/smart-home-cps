第二部分是基于 UDP 的可靠有序传输协议设计，展示自定义报文、ACK 机制、超时重传和按序交付。

我先用一句话概括：
“通过 TCP 流分析，我们能解释网络行为；通过 UDP 可靠协议设计，我们能在轻量场景下实现可控的可靠有序传输。”

一、需求与目标（约1分钟）
在 CPS 场景里，监测数据要可解释、可追溯，还要可靠、按序到达。
所以我给自己设定了两个目标：
1）离线解析 pcap，按四元组聚合 TCP 流，输出关键字段和载荷内容；
2）设计一个最小字段的 UDP 可靠有序协议，包含 ACK、超时重传、按序交付。

二、TCP 流分析（约4分钟）
我先讲 TCP 流分析的思路：
核心流程是“读 pcap -> 解析以太网/IP/TCP -> 校验长度 -> 按流聚合 -> 输出握手/终止与段信息”。

具体解析逻辑是这样：
- 逐包读取，先检查以太网头长度，确保最少 14 字节。
- 看以太网类型字段是否是 0x0800，只分析 IP 包。
- 解析 IP 头，取出 IHL 和 total length，验证包是否被截断。
- 只处理 TCP 协议（proto == 6）。
- 解析 TCP 头，读取 src/dst 端口、seq、ack、flags、window，计算 TCP 头长度，拿到 payload。
- 用四元组 src_ip:src_port-dst_ip:dst_port 作为流的标识，把每个段追加到对应流里。

这样做的意义是：
- 我们可以按流观察序号、确认号、窗口变化，了解流内的可靠传输过程；
- 也能把 payload 展示出来，辅助业务层理解；
- 更重要的是可以识别握手和终止过程，验证协议行为是否正常。

握手识别我用的是控制位判断：
- SYN、SYN|ACK、ACK 三个阶段按顺序出现就是三次握手。
- FIN 出现标识连接关闭。

输出层面，我会打印每段的 flags/seq/ack/window/len，并在有 payload 时展示内容。
这部分对应课题目标中的：
- TCP 报文段序号、确认号、控制位、窗口值变化规律；
- TCP 流中传输数据内容分析。

三、基于 UDP 的可靠有序传输设计（约4分钟）
第二部分是可靠传输协议设计。我选 UDP 作为承载，原因是它轻量、可控，便于自定义头部和策略。

我设计的自定义头部字段非常精简：
- seq：4 字节序号
- ack：4 字节确认号
- flags：1 字节，标记 DATA 或 ACK
- length：2 字节负载长度

打包逻辑是把这些字段按固定顺序打包，然后拼接 payload。
解析逻辑是先检查最小长度（11 字节），再按字段拆解。

发送端机制：
- 发送 DATA 包后等待 ACK；
- 如果超时，就重传；
- 收到 ACK 且 ack==seq 才认为成功，然后 seq 自增。

接收端机制：
- 收到 DATA 包立即回 ACK；
- 把乱序包放进 buffer；
- 从 expected 序号开始连续输出，实现按序交付。

这套机制体现了三点可靠性关键：
1）ACK 确认：保证发送端知道对方已收到；
2）超时重传：保证丢包后能够恢复；
3）有序交付：保证应用层看到的数据是连续的。

我在实验中观察到：
- 发送端能输出 ACK 和必要的 RESEND；
- 接收端按序输出 seq=1,2,3... 的数据；
这说明有序交付是生效的。

四、实验步骤与结果（约1分钟）
TCP 流分析：

- 列出流并选择目标流；

UDP 可靠传输：

- 再启动发送端：reliable_udp.py send；
- 观察 ACK 与重传行为，以及接收端的按序输出。
