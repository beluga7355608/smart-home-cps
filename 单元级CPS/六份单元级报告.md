## 报告1：ARP 发送与解析（jz）

### 摘要
本报告围绕ARP帧结构的设计、构造与解析展开，给出字节级字段组织方式，并在Windows环境下通过SendARP完成ARP解析验证，实现ARP层数据结构与解析流程的对照。

### 前言
ARP是以太网中IP到MAC解析的核心协议。单元级CPS通过可控的帧构造与解析，帮助理解ARP字段语义与实际网络解析过程。

### 需求分析
- 能构造完整Ethernet+ARP帧并输出字节序列。
- 能解析ARP帧/负载，提取源/目的MAC与IP。
- 在Windows环境下进行ARP解析验证。

### 方案论证
- 采用结构化打包方式实现字段控制，避免依赖第三方库。
- Windows原生限制无法直接发送以太网帧，因此采用SendARP触发解析，以“帧构造+解析”实现教学验证。

### 软件设计（最简实现）
构造与解析核心逻辑如下：

```python
eth = dst_mac + src_mac + struct.pack('!H', 0x0806)  # Ethernet头: 目的MAC+源MAC+类型(ARP)
arp = struct.pack('!HHBBH', 1, 0x0800, 6, 4, opcode) \  # ARP头: 硬件类型/协议类型/硬件长/协议长/操作码
    + src_mac + socket.inet_aton(src_ip) + dst_mac + socket.inet_aton(dst_ip)  # ARP地址字段
frame = eth + arp  # 完整Ethernet+ARP帧

htype, ptype, hlen, plen, opcode = struct.unpack('!HHBBH', arp[:8])  # 解析ARP头字段
src_ip = socket.inet_ntoa(arp[14:18])  # 解析源IP
dst_ip = socket.inet_ntoa(arp[24:28])  # 解析目的IP
```

### 实验与结果
- build：输入源/目的MAC与IP，输出十六进制帧。
- parse：输入十六进制帧，输出ARP字段信息。
- send：通过SendARP解析指定IP的MAC，验证解析流程。

### 总结
ARP模块完成了帧结构设计与解析验证，满足单元级CPS的“结构-过程-验证”闭环目标，并兼顾Windows环境约束。

## 报告2：TCP 发送与接收（rzh）

### 摘要
本报告实现TCP报文段结构的设计与“发送/接收演示”。通过构造TCP头并作为负载在TCP连接中传输，完成字段解析与数据接收验证。

### 前言
TCP提供可靠传输机制。单元级CPS需理解TCP报文段字段构成与接收端解析过程，以服务上层实时监测需求。

### 需求分析
- 设计TCP报文段核心字段结构。
- 具备发送与接收演示，能够解析发送的TCP头字段。
- 提供可操作的命令行运行方式。

### 方案论证
- 不直接发送原始TCP报文，避免系统权限与复杂校验问题。
- 使用普通TCP连接承载“构造的TCP头+数据”，实现结构化教学演示。

### 软件设计（最简实现）
构造TCP头并通过普通TCP连接发送：

```python
hdr = struct.pack('!HHLLBBHHH', src, dst, seq, ack, 0x50, flags, win, 0, 0)  # TCP头(20字节, offset=5)
s.sendall(hdr + payload)  # 发送TCP头+业务数据

src, dst, seq, ack, off, flags, win, chk, urg = struct.unpack('!HHLLBBHHH', data[:20])  # 解析TCP头字段
```

### 实验与结果
- server模式监听端口并解析接收的TCP头。
- client模式发送构造头与payload，服务端输出字段信息与payload内容。

### 总结
TCP模块完成了报文段结构设计与接收解析演示，为后续TCP流分析与可靠传输设计提供基础。

## 报告3：ICMP 活动主机发现（yzr）

### 摘要
本报告基于ICMP回送请求/应答实现活动主机扫描，包含ICMP报文结构构造、校验和计算与并发扫描流程。

### 前言
ICMP回送机制是常见的活性探测手段。单元级CPS通过实现ICMP扫描，提高对网络可达性与存活主机的感知能力。

### 需求分析
- 设计ICMP回送请求报文结构。
- 计算ICMP校验和，保证报文合法性。
- 支持IP范围扫描与并发提升效率。

### 方案论证
- 采用原始套接字发送ICMP报文，保证对协议字段的控制能力。
- 多线程扫描提高大范围扫描效率。

### 软件设计（最简实现）
回送请求与校验和核心代码：

```python
hdr = struct.pack('!BBHHH', 8, 0, 0, ident, seq)  # ICMP回送请求头(类型=8, 代码=0)
chk = checksum(hdr + payload)  # 计算校验和覆盖头+负载
pkt = struct.pack('!BBHHH', 8, 0, chk, ident, seq) + payload  # 填入校验和后生成最终报文
s = socket.socket(socket.AF_INET, socket.SOCK_RAW, socket.IPPROTO_ICMP)  # 原始套接字发送ICMP
s.sendto(pkt, (ip, 0))  # 发送到目标主机
```

### 实验与结果
- 指定起止IP与线程数进行扫描。
- 输出存活主机与RTT，统计总数。

### 总结
ICMP扫描模块完成从报文构造到并发探测的闭环验证，为系统级监测提供基础能力。

## 报告4：IP 流量监控统计（syz）

### 摘要
本报告实现按源IP统计IP包数量，完成“时间窗内包计数与结果输出”的监控任务，并给出CSV输出格式。

### 前言
在CPS场景中，实时流量统计是判断数据源活跃度的重要依据。单元级CPS通过IP流量监控形成源地址的统计视图。

### 需求分析
- 捕获IP数据包并解析源地址。
- 在指定时间窗内按源IP计数。
- 支持CSV输出与简要终端可视化。

### 方案论证
- 使用原始套接字捕获IP包，保证统计准确性。
- 采用计数器结构提升统计效率。

### 软件设计（最简实现）
按源IP计数的核心逻辑：

```python
sock = socket.socket(socket.AF_INET, socket.SOCK_RAW, socket.IPPROTO_IP)  # 创建原始IP套接字
sock.bind((local_ip, 0))  # 绑定本机IP接收所有IP包
sock.setsockopt(socket.IPPROTO_IP, socket.IP_HDRINCL, 1)  # 数据包含IP头
sock.ioctl(socket.SIO_RCVALL, socket.RCVALL_ON)  # 启用混杂模式抓包
counter = collections.Counter()  # 统计源IP次数
pkt = sock.recvfrom(65535)[0]  # 接收一个IP包
src = socket.inet_ntoa(pkt[12:16])  # 取IP头源地址字段
counter[src] += 1  # 计数加1
```

### 实验与结果
- 运行指定duration后生成CSV文件。
- 输出源IP计数排行与简单条形统计。

### 总结
该模块完成了源地址流量统计任务，为系统级CPS的流量感知与异常识别提供基础数据。

### 摘要
本报告实现IP头字段解析，包含版本、首部长度、标识、分片、TTL、协议号与源/目的地址等字段解析与日志记录。

### 前言
IP头解析是理解网络通信的基础能力。单元级CPS通过抓包与解析强化对网络协议字段的认识。

### 需求分析
- 捕获IP包并解析IP头字段。
- 打印解析结果并可选落盘日志。
- 兼容Windows原始套接字接收。

### 方案论证
- 采用IP_HDRINCL与混杂模式接收IP包。
- 字节切片+struct解包实现高效解析。

### 软件设计（最简实现）
IP头字段解析核心：

```python
ver_ihl = pkt[0]  # 版本(高4位)+首部长度(低4位)
ihl = (ver_ihl & 0x0F) * 4  # 首部长度(字节)
total_len = struct.unpack('!H', pkt[2:4])[0]  # 总长度
ttl = pkt[8]  # 生存时间TTL
proto = pkt[9]  # 上层协议号
src = socket.inet_ntoa(pkt[12:16])  # 源IP
dst = socket.inet_ntoa(pkt[16:20])  # 目的IP
```

### 实验与结果
- 执行指定count后输出解析结果。
- 生成日志文件保存抓包解析记录。

### 总结
IP解析模块实现了对IP头字段的可视化与记录，为后续流量统计与协议分析提供支撑。

## 报告6：网络数据分析 + 可靠传输协议设计（xyh）

### 摘要
本报告合并完成TCP流分析与基于UDP的有序传输协议设计：前者分析TCP序号、确认号、控制位与窗口变化；后者实现简单ACK与重传机制保证有序传输。

### 前言
网络数据分析用于理解传输行为，可靠传输协议设计用于满足监测数据“有序交付”的要求。两者共同服务系统级CPS的数据可靠性。

### 需求分析
- 从pcap中解析TCP流并输出控制位、序号、确认号与窗口值变化。
- 能展示TCP负载内容以辅助业务理解。
- 基于可靠传输机理设计简单协议，支持ACK、重传与有序交付。

### 方案论证
- pcap解析采用离线分析，避免实时抓包的不稳定因素。
- 可靠传输基于UDP实现，便于自定义协议字段与重传策略。

### 软件设计（最简实现）
TCP流解析与可靠UDP的最小逻辑：

```python
eth_type = struct.unpack('!H', pkt[12:14])[0]  # 以太网类型(0x0800为IP)
ihl = (pkt[14] & 0x0F) * 4  # IP头长度
src_ip = socket.inet_ntoa(pkt[26:30])  # IP源地址
dst_ip = socket.inet_ntoa(pkt[30:34])  # IP目的地址
src_port, dst_port, seq, ack, off_res, flags, win = struct.unpack('!HHLLBBH', pkt[14+ihl:14+ihl+16])  # TCP头字段

hdr = struct.pack('!IIBH', seq, ack, flags, len(payload))  # 自定义UDP头: 序号/确认号/标志/长度
s.sendto(hdr + payload, (host, port))  # 发送自定义可靠UDP包
seq, ack, flags, length = struct.unpack('!IIBH', data[:11])  # 解析可靠UDP头
```

### 实验与结果
- TCP流分析：列出流并选择目标流，输出控制位序列与payload内容。
- 可靠传输：发送端输出ACK/重传日志，接收端按序输出消息。

### 总结
该组合模块完成了“传输行为分析 + 可靠传输设计”的闭环，满足单元级CPS对数据可解释性与有序性要求。
