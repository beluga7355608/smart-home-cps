# 使用说明（中文）

## 1. 运行环境
- Windows 10/11
- Python 3.9+

## 2. 安装与启动

在项目根目录执行：

```powershell
python -m venv venv
venv\Scripts\Activate.ps1
pip install -r "智能家居CPS\requirements.txt"
python "智能家居CPS\server.py"
```

另开一个终端运行设备模拟器：

```powershell
python "智能家居CPS\device_sim.py"
```

可选：带参数启动（用于测试与对照实验）：

```powershell
# 设备数量、丢包、延迟、抖动、上报间隔
python "智能家居CPS\device_sim.py" --num-devices 20 --drop-rate 0.1 --delay-ms 100 --jitter-ms 50 --interval-min 3 --interval-max 6

# 固定随机种子，便于结果复现
python "智能家居CPS\device_sim.py" --num-devices 5 --seed 42
```

打开浏览器访问：

- 本机访问：http://127.0.0.1:5000
- 局域网访问：http://<本机IP>:5000

提示：
- 确保手机/其他电脑与服务器在同一局域网内。
- Windows 防火墙需放行 5000 端口（允许 Python/Flask 通过）。
- 页面顶部可设置访问口令，默认口令为 cps1234（可在 server.py 中修改 ACCESS_TOKEN，或设置环境变量 CPS_ACCESS_TOKEN）。

## 2.1 测试参数一览（设备模拟器）

可用参数：
- `--num-devices`：模拟设备数量（如 5/20/50）
- `--types`：设备类型列表（如 `temp,light,door,smoke,plug`）
- `--drop-rate`：丢包率（0-1）
- `--delay-ms`：固定额外延迟（毫秒）
- `--jitter-ms`：随机抖动延迟（毫秒）
- `--interval-min`：上报最小间隔（秒）
- `--interval-max`：上报最大间隔（秒）
- `--max-retry`：最大重试次数
- `--seed`：随机种子（用于复现实验）

## 3. 页面功能
- **设备列表**：显示在线设备、最后值、最后在线时间。
- **实时曲线**：每设备一张曲线图，支持缩放/拖拽。
- **历史回溯**：选择设备与时间范围，点击“回溯”加载历史数据。
- **下发命令**：在“操作”栏选择命令并发送。
- **场景模式**：回家/离家/夜间一键联动。
- **告警导出**：导出告警 CSV 文件。
- **网络指标**：展示平均时延、丢包率、校验错误、重复包等指标。
- **手动告警**：页面按钮一键触发门磁/烟雾告警（便于测试）。
- **告警开关**：先“开启告警”后，门磁/烟雾到阈值才会自动告警；关闭后不自动告警。
- **轮询实验**：页面顶部可启用轮询模式并设置间隔，用于与 WebSocket 对照测试。

## 3.1 轮询对照测试（页面内）

步骤：
1) 打开页面后点击“启用轮询实验”，设置轮询间隔（如 1000ms）。
2) 观察“轮询实验”统计区的请求数与时延。
3) 点击“停止轮询实验”，恢复 WebSocket 实时推送。

## 4. 可用命令

目前支持的命令如下（服务器端会校验）：

- `turn_on_light`：打开灯光
- `turn_off_light`：关闭灯光
- `turn_on_cooler`：打开冷却器
- `turn_off_cooler`：关闭冷却器
- `turn_on_plug`：打开插座
- `turn_off_plug`：关闭插座
- `alarm_on`：开启告警
- `alarm_off`：关闭告警
- `auto_on`：开启自动控制（仅 light/temp）
- `auto_off`：关闭自动控制（仅 light/temp）

说明：
- `temp` 设备允许 `turn_on_cooler/turn_off_cooler`（用于温控联动演示）。
- 不同类型设备只能执行自身允许的命令，跨设备命令会被服务端拒绝。

可通过接口检查覆盖：
- `GET /api/command-coverage`：查看每个设备是否有对应命令。

## 4.1 自动策略说明（服务端）

当设备上报数据时，服务端会按以下规则计算“最近命令”：

- **优先级**：若该设备有待执行的人工命令（页面下发/场景下发），先执行人工命令；否则执行自动策略。
- **灯光（light）/温度（temp）**：
	- 只有在“自动控制”开启时，才会根据阈值自动开关。
	- 关闭自动控制后，仅执行用户手动选择的开/关命令。
- **插座（plug）**：
	- 仅执行用户手动选择的开/关命令，不做自动控制。
- **门磁（door）**：
	- 告警开关关闭：不自动告警
	- 值 == 1（门打开）：`alarm_on`
	- 值 == 0（门关闭）：`alarm_off`
- **烟雾（smoke）**：
	- 告警开关关闭：不自动告警
	- 值 >= 50：`alarm_on`
	- 值 < 50：`alarm_off`

备注：
- 启动后默认**告警开启**，门磁/烟雾达到阈值会直接告警。
- 灯光/温度默认**自动控制开启**，可手动切换为“关闭自动”。
- “最近命令”会随着设备上报值变化而变化，这是正常现象。

## 5. 网络诊断工具（优化版）

### 5.1 ICMP 主机发现工具（icmp_scan.py）

**功能**：使用多线程并发 ICMP (ping) 快速扫描网络中的活跃主机。

**基本用法**：
```powershell
# 扫描 /24 子网
python "智能家居CPS\icmp_scan.py" 192.168.1.0/24

# 扫描 /25 子网（更小范围，更快）
python "智能家居CPS\icmp_scan.py" 192.168.1.0/25
```

**功能特性**：
- ✓ 多线程并发扫描（最多 100 个线程）
- ✓ 实时进度显示（逐行输出发现的主机）
- ✓ 延迟测量（毫秒级响应时间）
- ✓ 统计摘要（存活率、扫描速度等）

**输出示例**：
```
============================================================
ICMP 网络扫描 - 192.168.1.0/24
============================================================

  ✓ [  1] 存活: 192.168.1.1        延迟: 2.15ms
  ✓ [  2] 存活: 192.168.1.8        延迟: 1.89ms
  ✓ [  3] 存活: 192.168.1.105      延迟: 3.42ms

============================================================
统计信息:
  总扫描数:  254
  存活主机:  3 (1%)
  耗时:      5.23s
  平均速度:  48.6 个主机/秒
```

**性能对比**：
- 优化前（串行）：扫描 /24 需要约 300 秒
- 优化后（100 线程）：扫描 /24 只需约 5 秒
- **性能提升：60 倍**

### 5.2 TCP 连通性探测工具（tcp_probe.py）

**功能**：测试目标主机的端口是否开放、可达，支持批量扫描。

**用法示例**：
```powershell
# 探测单个端口
python "智能家居CPS\tcp_probe.py" 127.0.0.1 10001

# 探测端口范围
python "智能家居CPS\tcp_probe.py" 192.168.1.8 10000-10010

# 探测多个常见端口
python "智能家居CPS\tcp_probe.py" 127.0.0.1 22,80,443,3306,5000,10001
```

**功能特性**：
- ✓ 三种扫描模式（单端口、范围、列表）
- ✓ 并发扫描（最多 50 个线程）
- ✓ 详细连接状态（open/refused/timeout/error）
- ✓ 响应延迟测量（毫秒级）

**输出示例**：
```
开始扫描 127.0.0.1，共 6 个端口

============================================================
TCP 扫描结果 - 127.0.0.1
============================================================

统计信息:
  总扫描数:  6
  开放端口:  3
  关闭端口:  3

开放的端口:
  [OPEN    ] 端口  5000 - 响应时间: 0.45ms
  [OPEN    ] 端口 10001 - 响应时间: 0.87ms

所有结果详情:
  端口 22   : [REFUSED ] 错误: Connection refused
  端口 80   : [TIMEOUT ] 错误: Connection timeout
  端口 443  : [TIMEOUT ] 错误: Connection timeout
  端口 5000 : [OPEN    ] 延迟: 0.45ms
  端口 3306 : [REFUSED ] 错误: Connection refused
  端口 10001: [OPEN    ] 延迟: 0.87ms

============================================================

扫描耗时: 0.93s
```

**原理说明**：
- 工具尝试与每个端口建立 TCP 连接（三次握手）
- 连接成功 = 端口开放，立即关闭连接
- 连接被拒绝 = 端口关闭
- 连接超时 = 端口被过滤或主机不可达

### 5.5 完整诊断工作流

按以下顺序使用工具可形成"逐层诊断"的完整过程：

**场景**：需要排查与某个网络地址的连接问题

1. **第一步 - 发现活跃主机**
   ```powershell
   python "智能家居CPS\icmp_scan.py" 192.168.1.0/24
   ```
   找到目标主机 IP（如 192.168.1.8）

2. **第二步 - 探测服务端口**
   ```powershell
   python "智能家居CPS\tcp_probe.py" 192.168.1.8 22,80,3306,5000,10001
   ```
   确认哪些端口开放、响应延迟如何

3. **第三步 - 查看本地连接状态**
   ```powershell
   python "智能家居CPS\traffic_stats.py"
   ```
   观察是否已建立到目标的连接

4. **第四步 - 检查 ARP 缓存**
   ```powershell
   python "智能家居CPS\arp_table.py"
   ```
   确认 IP 地址与 MAC 地址的映射关系

### 5.6 与项目系统的集成

这些诊断工具专门为智能家居 CPS 系统设计：

- **icmp_scan.py**：发现局域网中的设备
- **tcp_probe.py**：验证设备的数据端口（10001）是否可达
- **traffic_stats.py**：监控系统与设备之间的连接状态
- **arp_table.py**：排查设备的 MAC 地址问题

**典型诊断场景**：
```powershell
# 1. 发现网络中运行的设备
python "智能家居CPS\icmp_scan.py" 192.168.1.0/24

# 2. 验证发现的设备的数据端口（10001）是否开放
python "智能家居CPS\tcp_probe.py" 192.168.1.8 10001,10000

# 3. 启动系统
python "智能家居CPS\server.py"

# 4. 启动设备模拟器或真实设备

# 5. 监控连接状态
python "智能家居CPS\traffic_stats.py"
```

设备模拟器会打印当前状态，如：

```
[RESP] light01 cmd=turn_on_light state={'light': 'on'}
```

## 6. 常见问题

1) 页面一直显示“WebSocket: 连接中…”
- 请确认 `server.py` 已重启到最新版本，刷新页面（Ctrl+F5）。

2) 设备列表为空
- 确认 `device_sim.py` 正在运行，查看终端是否有 `[RESP]` 日志。
