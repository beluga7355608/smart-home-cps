# 设计文档：基于CPS的智能家居安全与节能协同管理系统（完善版）

1. 目标与范围
- 目标：构建可感知、可计算、可交互的单元级CPS原型，实现设备发现、可靠上报、策略决策、命令下发、可视化监控、历史回溯。
- 范围：两室一厅家庭场景，模拟温度与光照设备，中央服务器负责策略与展示。

2. 总体架构
- 三层：感知层（模拟设备）— 网络层（局域网，UDP/TCP）— 控制与应用层（中央服务器 + Web 可视化）。
- 控制闭环：设备上报 -> 服务器分析 -> 策略生成 -> 指令下发 -> 设备执行。

3. 协议与实现选择（思辨）
- 发现：原型使用 UDP 广播（跨平台、低门槛），对比 ICMP/ARP：ARP需同网段且偏二层，ICMP需权限与易被屏蔽；UDP利于模拟与快速扩展。
- 数据：使用 TCP 上报关键数据与接收控制指令，保证可靠、有序与简化服务器处理逻辑。
- 实时推送：使用 WebSocket（Socket.IO）双向通信，减少轮询开销并支持主动推送。
- 应用层可靠性扩展：在负载中加入 `seq/send_ts/checksum`，用于丢包估计、端到端时延估计与差错检测。

4. 需求与约束
- 功能需求：自动发现设备、上报数据、策略联动、告警提示、Web 可视化、历史回溯与图表缩放。
- 非功能需求：响应<2s、丢包容忍、可扩展到10+设备、可在普通实验电脑运行。

5. 关键模块设计
- 设备发现：设备周期广播 JSON（device_id/type），服务器监听并更新在线列表。
- 数据上报：TCP 发送 JSON（device_id/type/value/ts），服务器解析并写入SQLite。
- 策略引擎：示例规则，光照<30触发 `turn_on_light`；插座负载>55触发 `turn_off_plug`。
- 命令权限控制：按设备类型约束命令集合，防止设备执行非本类型命令（体现访问控制思想）。
- 可视化：WebSocket 推送实时数据，前端图表支持缩放/回溯与命令下发。
- 协议实验脚本：ICMP 扫描、TCP 连通性探测、连接统计。
- ARP 观察：使用 `arp -a` 或脚本查看局域网邻居信息。

6. 数据模型与存储
- 设备内存结构：device_id, type, ip, last_seen, last_value, pending_cmd。
- 历史表：readings(device_id, device_type, value, ts)，用于回溯查询与趋势分析。
- 网络指标：每设备统计接收包数、估计丢包数、校验错误数、平均时延、重复包数。

7. 安全与异常处理
- 基础控制：命令合法性校验，异常数据记录。
- 连接失败处理：设备重试，上线/掉线由 last_seen 超时判定。

8. 测试计划
- 功能测试：发现、上报、策略、命令下发、WebSocket 推送。
- 性能测试：多设备并发上报、历史回溯请求压力。
- 协议测试：ICMP 扫描结果、TCP 探测结果、连接统计变化。
- 验收用例：见 `使用说明.md` 中 TC-01 ~ TC-10。
- 网络可靠性测试：通过模拟丢包观察丢包率变化、checksum 错误告警、seq 连续性与重传日志。
