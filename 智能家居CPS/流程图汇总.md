# 智能家居CPS 流程图汇总

## 1. 系统主流程图

```mermaid
flowchart TD
    A[启动 server.py] --> B[初始化SQLite与Flask-SocketIO]
    B --> C[并发启动三个线程]
    C --> C1[UDP发现监听:10000]
    C --> C2[TCP数据服务:10001]
    C --> C3[设备超时清理]
    C --> D[启动Web服务:5000]

    E[启动 device_sim.py] --> E1[每设备线程1: UDP广播发现]
    E --> E2[每设备线程2: 周期生成传感数据]

    E1 --> C1
    C1 --> F[更新设备在线状态]
    F --> G[SocketIO推送设备列表到前端]

    E2 --> H[构造上报JSON: device_id/type/value/seq/send_ts/checksum]
    H --> I[TCP发送到服务器]
    I --> C2

    C2 --> J{校验与统计}
    J --> J1[checksum校验]
    J --> J2[seq统计丢包/重复]
    J --> J3[计算时延]

    J3 --> K[写入SQLite历史readings]
    K --> L[策略引擎apply_policy]
    L --> M{是否有人工待执行命令}
    M -->|有| N[优先执行人工命令]
    M -->|无| O[按设备类型自动策略]

    O --> O1[light/temp: 自动开关]
    O --> O2[door/smoke: 告警策略]
    O --> O3[plug: 仅手动]

    N --> P[返回响应JSON: cmd/ack_seq/server_ts]
    O1 --> P
    O2 --> P
    O3 --> P

    P --> Q[设备执行命令并更新本地状态]
    Q --> E2

    K --> R[SocketIO推送reading/devices/alarm]
    R --> S[index.html实时刷新设备/曲线/告警/网络指标]

    D --> T[前端REST接口调用]
    T --> T1[/api/devices]
    T --> T2[/api/history]
    T --> T3[/api/network-stats]
    T --> T4[/api/command与场景接口]
```

## 3. 网络诊断工具链流程图

```mermaid
flowchart LR
    A[icmp_scan.py: ICMP并发扫描网段] --> B{发现活跃主机?}
    B -->|是| C[tcp_probe.py: 扫描目标端口]
    B -->|否| Z[结束: 先排查网络连通]

    C --> D{端口状态}
    D -->|open| E[服务可达并记录延迟]
    D -->|refused/timeout| F[服务未就绪或被过滤]

    E --> G[traffic_stats.py: 查看本机连接状态]
    F --> G

    G --> H[arp_table.py: 检查IP-MAC映射]
    H --> I[形成定位结论: 主机可达性/端口可达性/连接状态/ARP映射]
```

## 4. 闭环时序图（设备上报-策略回执-前端展示）

```mermaid
sequenceDiagram
    participant Dev as 设备(device_sim.py)
    participant Srv as 服务端(server.py)
    participant DB as SQLite
    participant UI as 前端(index.html)

    loop 每10秒
        Dev->>Srv: UDP discovery(device_id,type)
        Srv-->>UI: SocketIO devices(在线设备)
    end

    loop 每5-10秒
        Dev->>Srv: TCP上报(value,seq,send_ts,checksum)
        Srv->>Srv: 校验checksum/统计丢包重复/计算时延
        Srv->>DB: insert readings
        Srv->>Srv: apply_policy(人工优先,否则自动)
        Srv-->>Dev: 响应cmd + ack_seq + server_ts
        Dev->>Dev: 执行命令并更新状态
        Srv-->>UI: SocketIO reading/devices/alarm
    end

    UI->>Srv: REST查询(/api/history,/api/network-stats)
    Srv-->>UI: JSON数据
```
